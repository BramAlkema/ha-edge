Enable Local Fulfillment (Recommended)
Faster responses - Google devices talk directly to HA on your LAN
  1. In Developer Console, go to your project
  2. Go to your Cloud-to-cloud integration
  3. Enable Local Home SDK
  4. For Node.js and Chrome, enter this URL:
JavaScript URL (for both Node and Chrome):
{{ state.server_url }}/local-sdk/app.js
  1. Under Device scan configuration, click Add
  2. Select MDNS and enter:
mDNS service name:
_home-assistant._tcp.local

Name:
*\._home-assistant\._tcp\.local
  1. Check Support local query
  2. Click Save
  3. Wait 30 minutes or restart your Google devices
  4. Say "OK Google, sync my devices"
Why? Local fulfillment sends commands directly over your LAN via mDNS, skipping the cloud round-trip. Much faster response times!

Add Dashboard Card (Optional)
Monitor tunnel status from your dashboard

Sensors are automatically created:

  • sensor.gcp_tunnel_status - healthy/disconnected
  • binary_sensor.gcp_tunnel_connected - on/off

Add this card to your dashboard:

type: entities
title: GCP Tunnel
entities:
  - entity: sensor.gcp_tunnel_status
    name: Status
  - entity: binary_sensor.gcp_tunnel_connected
    name: Connected
  - entity: binary_sensor.gcp_tunnel_report_state
    name: Report State
Tip: Restart HA Core once to load the new sensors.
